# データ競合を発生させる

iOSアプリでは、複数のタスクを使って効率的にプログラムを実行していくことがよくあります。このような処理をマルチタスクと言いますが、このときに複数のタスクから同じデータにアクセスするとデータが壊れることがあります。この現象を `データ競合 (Data race)` と呼びます。

データ競合が発生する原因は「複数のタスクでひとつのデータを同時操作しているときに、どこかで書込操作が行われること」です。そうなると、書込操作が完全に終わらないうちに別のタスクが読み込んでしまい、予期しないデータが得られることになります。この問題はたまたま同じタイミングで読み書きが発生したときだけに起こるため、再現性が低く、想定しないデータによって深刻な影響が及ぶこともあって厄介です。

Swift には、データ競合を回避するための幾つかの方法が用意されています。  
ここではそれを学んでいきますが、データ競合に馴染みがないと、どのような問題が起こるかイメージしにくいところがあるので、まずは実際にデータ競合を発生させてみましょう。

## 課題

２週間分の天気をそれぞれ `sync var` を用いて取得します。取得した天気は配列に入れ、全部が揃った段階でテーブルビューに表示してください。

今回は敢えてデータ競合を起こしたいので `Swift 言語モード` を `5` に設定した上で、`厳密な並行安全性チェック` を最小限 (`minimal`) に設定してください。データの取得は `Dispatch` による並行処理で取得し、ロックなどの排他制御は取らないようにします。そうして実際に、期待したのと違う結果が得られることを確認できたら、本課題はクリアです。

*   ２週間分の天気を `sync var` で取得します。
*   それぞれの天気は `Dispatch` を使った並行処理で取得します。
*   すべての天気が取得できたら、テーブルビューに一覧表示します。

この課題では、天気を取得する際にエラーが発生することがあります。２週間分の天気を取得する中のどれかでエラーが発生したときに、そのエラーをどう扱うのが良いか、考えて実装してみましょう。
