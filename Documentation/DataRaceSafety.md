# データ競合安全を図る

前の課題で、`Dispatch` を使った並行処理でデータ競合が意図的に発生することを確認しました。データ競合は、再現性が低くデバッグが困難な問題を引き起こす厄介な問題です。

Swift は 5.5 から、データ競合を防ぐための Swift Concurrency という仕組みが導入されました。これは並行処理を行うときに、書き込みを伴う同時アクセスを発生させないことでデータ競合を防止します。

ここではその Swift Concurrency を使って、前の課題で発生したデータ競合の問題を解決してみましょう。

## 課題

前の課題で作成した、２週間分の天気を並行処理で取得し、配列に格納するコードを修正します。

Swift Concurrency を導入して、データ競合が発生しないようにしてください。
データ競合が発生せず、期待通りに２週間分の天気が表示されることを確認できたら、本課題はクリアです。

## ヒント
Swift Concurrency でデータ競合を防止する方法はいくつかありますが、ここでは `withTaskGroup` を使って実現することを考えてみましょう。

余力があれば Swift Concurrency のアクターを使って実現する方法も考えてみるのも良いでしょう。
